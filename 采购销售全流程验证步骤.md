# 采购销售全流程验证步骤

## 1. SQL 导入顺序
1. 导入基础结构与基础数据：`sql/01_schema.sql`、`sql/02_seed_data.sql`。
2. 导入本次流程演示数据：`sql/04_purchase_sales_flow_demo.sql`。
3. 若需要报表演示数据，再导入：`sql/03_demo_report_data.sql`。

说明：`sql/04_purchase_sales_flow_demo.sql` 已包含幂等写法，可重复执行，不会重复插入同一演示单据。

## 2. 采购流程验证
1. 进入“采购管理”，确认列表存在以下状态样例：`DRAFT`、`SUBMITTED`、`APPROVED`、`RECEIVED`。
2. 新建采购单：选择供应商/仓库，添加至少一条水果明细，保存后列表自动刷新。
3. 对草稿单点击“提交”，状态应变为“已提交”。
4. 对已提交单点击“审核”，状态应变为“已审核”。
5. 对已审核单点击“收货”，在弹窗中仅填写部分数量后提交：
   - 状态保持“已审核”；
   - 明细 `已收数量` 增加，`待收数量` 减少；
   - 库存批次数量同步增加。
6. 再次收货直至全部收完：状态应自动变为“已入库”。

## 3. 销售流程验证
1. 进入“销售管理”，确认列表存在以下状态样例：`DRAFT`、`SUBMITTED`、`APPROVED`、`SHIPPED`。
2. 新建销售单：选择客户/仓库，添加水果明细，保存后列表自动刷新。
3. 对草稿单点击“提交”，状态应变为“已提交”。
4. 对已提交单点击“审核”，状态应变为“已审核”。
5. 对已审核单点击“出库”，在弹窗中仅填写部分数量后提交：
   - 状态保持“已审核”；
   - 明细 `已出数量` 增加，`待出数量` 减少；
   - 库存按 FEFO 规则自动扣减。
6. 继续出库直至全部完成：状态应自动变为“已出库”。

## 4. 库存不足拦截验证
1. 对任意“已审核”销售单，在出库弹窗中填入超过待出数量或超过当前可用库存的数量。
2. 提交后应返回失败提示，不产生部分成功。
3. 回查销售单和库存，数据应保持操作前状态（事务回滚）。

## 5. 前后端编译检查
1. 后端执行：`mvn -q -DskipTests compile`
2. 前端执行：`npm run build`
3. 两端均通过即说明本次改造无编译回归。
