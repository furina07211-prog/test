# 数据库配置说明

本文档用于说明 `fruit_warehouse_manage` 项目的数据库固定配置与导入步骤。

## 1. 固化连接信息
后端配置文件：`backend/src/main/resources/application-dev.yml`

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/fruit_warehouse_manage?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true&useSSL=false
    username: root
    password: 123456
    driver-class-name: com.mysql.cj.jdbc.Driver
```

## 2. 初始化脚本（当前可用）
仅使用以下两个脚本：
1. `sql/_sanitized_schema.sql`
2. `sql/_seed_ascii.sql`

## 3. 导入方式（MySQL CLI）
```bash
mysql -h 127.0.0.1 -P 3306 -u root -p123456 -e "CREATE DATABASE IF NOT EXISTS fruit_warehouse_manage DEFAULT CHARSET utf8mb4;"
mysql -h 127.0.0.1 -P 3306 -u root -p123456 fruit_warehouse_manage < sql/_sanitized_schema.sql
mysql -h 127.0.0.1 -P 3306 -u root -p123456 fruit_warehouse_manage < sql/_seed_ascii.sql
```

## 4. 验证连接
在 `backend` 目录执行：
```bash
mvn spring-boot:run
```

启动后可访问：
- `http://localhost:8080/doc.html`
- `http://localhost:8080/swagger-ui/index.html`

## 5. 常见错误
1. `Access denied for user`：检查 MySQL root 密码是否为 `123456`。
2. `Unknown database`：先执行建库命令，再导入脚本。
3. 时区报错：确认 JDBC URL 含 `serverTimezone=Asia/Shanghai`。
