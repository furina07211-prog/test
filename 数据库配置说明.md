# 数据库配置说明

本文档用于说明 `fruit_warehouse_manage` 项目的 MySQL 数据库连接方式、初始化步骤和常见配置项。

## 1. 推荐环境
- 数据库版本：MySQL 8.x
- 推荐版本：MySQL 8.4.x LTS
- 字符集：`utf8mb4`
- 排序规则：`utf8mb4_0900_ai_ci`
- 时区：`Asia/Shanghai`

## 2. 当前后端连接配置
当前项目后端数据库配置文件位置：
- `backend/src/main/resources/application-dev.yml`

当前默认配置如下：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/fruit_warehouse_manage?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true&useSSL=false
    username: root
    password: 123456
    driver-class-name: com.mysql.cj.jdbc.Driver
```

## 3. 各配置项说明
- `url`：JDBC 连接地址，默认连接本机 `3306` 端口下的 `fruit_warehouse_manage` 数据库。
- `username`：数据库用户名，默认是 `root`。
- `password`：数据库密码，默认示例值是 `123456`，请按本机实际情况修改。
- `driver-class-name`：MySQL 8 驱动类，固定为 `com.mysql.cj.jdbc.Driver`。

连接串中的参数说明：
- `useUnicode=true`：启用 Unicode。
- `characterEncoding=utf8`：设置连接编码。
- `serverTimezone=Asia/Shanghai`：避免时区导致的时间偏移问题。
- `allowPublicKeyRetrieval=true`：允许获取公钥，适配部分本地 MySQL 环境。
- `useSSL=false`：本地开发阶段关闭 SSL，减少连接问题。

## 4. 数据库初始化步骤
建议按以下顺序执行 SQL 文件：

1. `sql/init_schema.sql`
2. `sql/test_data.sql`

如果你已经执行过旧版本脚本，也可以先清空数据库后重新导入，避免主键和测试数据重复。

## 5. 初始化方式示例
### 方式一：MySQL 命令行导入
```bash
mysql -u root -p
source D:/IDEAWorkspace/fruit_warehouse_manage/sql/init_schema.sql;
source D:/IDEAWorkspace/fruit_warehouse_manage/sql/test_data.sql;
```

### 方式二：Navicat / DataGrip / IDEA Database 导入
- 先连接本地 MySQL。
- 新建查询窗口。
- 先执行 `init_schema.sql`。
- 再执行 `test_data.sql`。

## 6. 本机配置不一致时如何修改
如果你的数据库用户名、密码或端口不是默认值，请修改以下内容：

文件：`backend/src/main/resources/application-dev.yml`

常见修改示例：

### 修改端口为 3307
```yaml
url: jdbc:mysql://localhost:3307/fruit_warehouse_manage?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true&useSSL=false
```

### 修改用户名和密码
```yaml
username: your_username
password: your_password
```

### 修改数据库名
如果你希望使用其他数据库名，例如 `fruit_manage_demo`，需要同时修改：
- `application-dev.yml` 中的 JDBC 地址
- `sql/init_schema.sql` 中的数据库名

## 7. 验证是否连接成功
完成配置后，在 `backend` 目录执行：

```bash
mvn spring-boot:run
```

如果启动成功，说明数据库连接正常。

也可以通过以下方式进一步验证：
- 访问后端接口文档页面 `http://localhost:8080/doc.html`
- 调用登录接口 `/api/auth/login`
- 查看水果、客户、销售历史等列表接口是否有返回数据

## 8. 常见问题
### 8.1 报错：Unknown database 'fruit_warehouse_manage'
原因：数据库还未创建。
解决：先执行 `sql/init_schema.sql`。

### 8.2 报错：Access denied for user
原因：用户名或密码错误。
解决：检查 `application-dev.yml` 中的 `username` 和 `password`。

### 8.3 报错：The server time zone value is unrecognized
原因：数据库时区设置不一致。
解决：确认 JDBC 地址中包含：
```text
serverTimezone=Asia/Shanghai
```

### 8.4 导入测试数据时报外键错误
原因：没有先执行建表脚本。
解决：必须先执行 `init_schema.sql`，再执行 `test_data.sql`。
