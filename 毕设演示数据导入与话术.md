# 毕设演示数据导入与话术

## 一、导入目标
本说明对应脚本：`sql/06_graduation_demo_full_data.sql`。  
脚本采用“幂等增量”策略：仅写入演示专用ID区间（`860000~899999`），可重复执行，不影响非演示业务数据。

## 四、导入后核验 SQL（可直接复制）
```sql
USE fruit_warehouse_manage;

-- 1) 账号与角色
SELECT id, role_code, role_name FROM sys_role WHERE id IN (1,2,3);
SELECT id, username, status FROM sys_user WHERE id IN (1,2,3);
SELECT user_id, role_id FROM sys_user_role WHERE user_id IN (1,2,3) ORDER BY user_id;

-- 2) 主数据数量
SELECT COUNT(*) AS fruit_count FROM fruit_info WHERE id BETWEEN 861100 AND 861199;
SELECT COUNT(*) AS supplier_count FROM supplier WHERE id BETWEEN 861300 AND 861399;
SELECT COUNT(*) AS customer_count FROM customer WHERE id BETWEEN 861400 AND 861499;
SELECT COUNT(*) AS warehouse_count FROM warehouse WHERE id BETWEEN 861200 AND 861299;

-- 3) 库存批次与场景
SELECT COUNT(*) AS batch_count FROM inventory_batch WHERE id BETWEEN 862000 AND 862099;
SELECT COUNT(*) AS near_expire_count
FROM inventory_batch
WHERE id BETWEEN 862000 AND 862099
  AND status = 'IN_STOCK'
  AND expiration_date <= DATE_ADD(CURDATE(), INTERVAL 3 DAY);
SELECT COUNT(*) AS low_stock_fruit_count
FROM (
  SELECT fi.id
  FROM fruit_info fi
  LEFT JOIN inventory_batch ib ON ib.fruit_id = fi.id AND ib.status = 'IN_STOCK'
  WHERE fi.id BETWEEN 861100 AND 861199
  GROUP BY fi.id, fi.safe_stock_qty
  HAVING COALESCE(SUM(ib.available_qty), 0) < fi.safe_stock_qty
) t;

-- 4) 采购/销售单据分布
SELECT COUNT(*) AS purchase_30d
FROM purchase_order
WHERE id BETWEEN 870000 AND 870999
  AND order_date >= DATE_SUB(CURDATE(), INTERVAL 30 DAY);
SELECT order_status, COUNT(*) AS cnt
FROM purchase_order
WHERE id BETWEEN 870000 AND 870999
GROUP BY order_status;

SELECT COUNT(*) AS sales_30d
FROM sales_order
WHERE id BETWEEN 880000 AND 880999
  AND order_time >= DATE_SUB(NOW(), INTERVAL 30 DAY);
SELECT order_status, COUNT(*) AS cnt
FROM sales_order
WHERE id BETWEEN 880000 AND 880999
GROUP BY order_status;

-- 5) 近3个月历史销量连续性
SELECT DATE(order_time) AS dt, COUNT(*) AS order_cnt, ROUND(SUM(total_amount), 2) AS amount
FROM sales_order
WHERE id BETWEEN 880000 AND 880999
  AND order_time >= DATE_SUB(NOW(), INTERVAL 90 DAY)
GROUP BY DATE(order_time)
ORDER BY dt;

-- 6) 预警可见性
SELECT COUNT(*) AS unhandled_alerts
FROM inventory_alert
WHERE id BETWEEN 896000 AND 896999
  AND alert_status = 'UNHANDLED';
```

## 六、毕设标准演示流程（口播话术版）
1. 登录（系统管理员）
   - 话术：这是基于 SpringBoot + Vue 的水果批发仓库管理系统，先用管理员账号进入全局视角。
2. Dashboard 数据看板
   - 话术：看板展示今日采购、今日销售、库存预警和品类占比，数据来自真实业务单据聚合。
3. 库存管理
   - 话术：这里能看到正常库存、临期批次与低库存风险，系统会自动生成预警。
4. 采购管理
   - 话术：演示采购单从草稿到提交、审核、收货入库的闭环，收货后库存会自动累加。
5. 销售管理
   - 话术：演示销售单从草稿到审核、出库的流程，系统按库存扣减并保留状态追踪。
6. AI 智能助手
   - 话术：AI 支持自然语言查询预警、销量排行和报表分析，涉及写操作时必须二次确认，避免误操作。
7. 总结
   - 话术：系统实现了“主数据-进销存-看板-AI分析”的完整业务链路，适合毕业设计展示与答辩。

## 七、常见问题处理
1. 如果导入报“字段不存在 shipped_qty”：请确认已使用 MySQL 8.x，脚本已包含兼容补丁。
2. 如果导入报“外键约束失败”：请严格按“00 -> 01 -> 04 -> 06”顺序执行。
3. 如果页面无数据：优先执行本文件中的核验 SQL，确认对应ID区间记录已成功写入。
