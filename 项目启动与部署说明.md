# 项目启动与部署说明

## 1. 环境要求
- JDK 21
- Maven 3.8+
- Node.js 18+（建议 20/22）
- MySQL 8.x
- Python 3.8+（仅 AI 算法脚本需要）

## 2. 固化配置（无需手改）
- 后端数据库：`localhost:3306/fruit_warehouse_manage`
- 后端账号：`root / 123456`
- 后端端口：`8080`
- 前端端口：`5173`
- 前端代理：`/api -> http://localhost:8080`

## 3. 数据库初始化
在项目根目录执行（按顺序）：

```bash
mysql -h 127.0.0.1 -P 3306 -u root -p123456 -e "CREATE DATABASE IF NOT EXISTS fruit_warehouse_manage DEFAULT CHARSET utf8mb4;"
mysql -h 127.0.0.1 -P 3306 -u root -p123456 fruit_warehouse_manage < sql/_sanitized_schema.sql
mysql -h 127.0.0.1 -P 3306 -u root -p123456 fruit_warehouse_manage < sql/_seed_ascii.sql
```

## 4. 启动后端
```bash
cd backend
mvn -q -DskipTests compile
mvn spring-boot:run
```

接口文档：
- Swagger UI：`http://localhost:8080/swagger-ui/index.html`
- Knife4j：`http://localhost:8080/doc.html`

## 5. 启动前端
```bash
cd frontend
npm install --cache .npm-cache
npm run dev
```

前端访问：`http://localhost:5173`

## 6. 登录账号
- 管理员：`admin / 123456`
- 仓库管理员：`warehouse / 123456`
- 销售员：`sales / 123456`

## 7. 常见问题
1. 启动报数据库连接失败：确认 MySQL 已启动，且 root 密码为 `123456`。
2. 前端请求 404/代理失败：确认后端已启动并监听 `8080`。
3. 端口占用：释放 `8080/5173` 后重启。
