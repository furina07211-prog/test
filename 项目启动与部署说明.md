# 项目启动与部署说明

## 环境要求
- JDK 21，Maven 3.8.8（已在机器 D:\develop\apache-maven-3.8.8）
- Node 22 + npm（前端 Vite）
- Python 3.8（运行 ai-algorithm 脚本）
- MySQL 8.4+，字符集 utf8mb4

## 数据库初始化
1. 创建库：`CREATE DATABASE fruit_warehouse_manage DEFAULT CHARSET utf8mb4;`
2. 依次执行脚本（在 `sql/` 目录）：
   - `00_init_db.sql`
   - `01_schema.sql`
   - `02_seed_data.sql`
   - `03_demo_report_data.sql`
   - `04_menu_init.sql`
   - `05_perf_indexes.sql`（新增高频查询索引，MySQL 8+ 支持 IF NOT EXISTS）

## 后端启动（Spring Boot 3.2.5）
1. 配置数据库连接：编辑 `backend/src/main/resources/application-dev.yml`，填入本地 `url/username/password`。
2. 编译：`cd backend && mvn -q -DskipTests compile`
3. 启动：`mvn spring-boot:run`（默认端口 8080）。
4. 接口文档：
   - Swagger UI: `http://localhost:8080/swagger-ui/index.html`
   - Knife4j: `http://localhost:8080/doc.html`

## 前端启动（Vue 3 + Vite）
1. `cd frontend`
2. 安装依赖：`npm install --cache .npm-cache`
3. 开发模式：`npm run dev`（默认 5173，已配 axios base `/` 透传到后端 /api）
4. 生产构建：`npm run build`

## AI 脚本与预测
1. `cd ai-algorithm`
2. 安装依赖：`pip install -r requirements.txt`
3. 单次预测示例：
   ```bash
   echo '{"fruitId":101,"days":7,"model":"prophet"}' | python scripts/predict_cli.py forecast
   ```
4. 采购建议示例：
   ```bash
   echo '{"predictedDailyQty":20,"leadTimeDays":2,"safeStockQty":30,"currentStockQty":10,"inTransitQty":5}' | python scripts/predict_cli.py optimize
   ```

## 常见问题
- CORS：已在 `CorsConfig` 放开 `*` 域名，若部署到特定域可改为白名单。
- Token：前端在请求头自动附加 `Authorization: Bearer <token>`，后端 JWT 解析失败会返回 401。
- 接口文档不可访问：确认应用启动且访问 `http://localhost:8080/doc.html`。
- 预测超时：可在 `application.yml` 调整 `fruit.algorithm.timeout-seconds` 或 `python-command` 路径。
