# 问题修复记录（最终版，2026-03-27）

1. 登录与权限角色映射修复（P0）
- 修复点：`UserMapper` 增加联表查角色，`User.roleCode` 设为非持久字段，`UserServiceImpl.getByUsername` 切换联表查询。
- 结果：JWT 角色稳定，`@PreAuthorize` 正常。

2. 明文密码兼容与自动升级（P0）
- 修复点：`AuthController` 支持明文兼容登录，并自动升级为 BCrypt。
- 结果：默认账号可直接登录，且逐步完成加密迁移。

3. Mapper 扫描冲突修复（P0）

4. 基础资料接口补齐（P1）
- 修复点：新增分类/仓库实体、Mapper、Service、Controller CRUD 接口。
- 结果：前端基础资料页接口全部可达。

5. 前端登录与 API 对齐（P0/P1）
- 修复点：`stores/user.js` 修正 Result 包装解析；`api/modules.js` 补齐仓库 API。
- 结果：token 可正确落库，仓库操作可调用。

6. 采购收货失败修复（P0）
- 问题：`inventory_batch.source_type` 非空字段未写入。
- 修复点：`PurchaseServiceImpl` 在收货入库时补齐 `sourceType/sourceId`。
- 结果：采购链路可完整流转到 `RECEIVED`。

7. AI 意图识别失效修复（P1）
- 问题：词表编码污染导致未命中本地工具查询。
- 修复点：`AiIntentServiceImpl` 重写意图词表（中英文）并统一小写匹配。
